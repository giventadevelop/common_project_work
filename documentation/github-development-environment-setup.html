GitHub Development Environment Setup

Playbook distilled from the local github-details.txt notes. Covers Git Bash, Personal Access Tokens (PAT), cloning, pushing from a new or existing project, Eclipse import, and GitHub CLI. No passwords, tokens, or account secrets are stored in this file.

Security: Store GitHub usernames, passwords, and PAT values only in a password manager or Windows environment variables on your machine — never in git-tracked files. If credentials were ever pasted into notes or docs, rotate the PAT in GitHub → Settings → Developer settings → Personal access tokens.

Accounts, email & PAT setup

Git identity (set once per machine — use your real values locally, not in git):

  • GitHub username: YOUR_GITHUB_USERNAME
  • Git commit email: your-email@example.com
  • HTTPS authentication: use a PAT as the password when prompted (not your GitHub account password).

Create a Classic PAT

  1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic).
  2. Generate new token with at least repo scope (and workflow if needed).
  3. Copy the token once (format ghp_…) and store it in your password manager.
Where PATs are used (locally only):
  • Windows environment variable GITHUB_PERSONAL_ACCESS_TOKEN or GIT_PAT_TOKEN
  • Git credential manager / HTTPS prompts (paste PAT when asked for password)
  • IDE integrations (IntelliJ, Claude Desktop config, MCP tools such as Byterover — store in local config files that are not committed)

Configure Git user (one-time)

Git global config
Customize parameters:
git config --global user.name "Your Name" && git config --global user.email "your-email@example.com"
Windows single-line:
git config --global user.name "Your Name" && git config --global user.email "your-email@example.com"

Video reference for PAT instead of password: GitHub token usage (YouTube) (based on related tutorial).

Environment variables

Set these in Windows System Environment Variables or your user profile — not in repository files.

VariablePurpose
GITHUB_PERSONAL_ACCESS_TOKENUsed by tools and scripts that read a standard GitHub PAT env var.
GIT_PAT_TOKENAlternate name used in some local configs; same value as your PAT.
Never commit .env, cipher.yml, or JSON config files containing ghp_… tokens. Add those paths to .gitignore.
Use Git Bash on Windows

Prefer Git Bash over CMD or PowerShell for git commands when you see HttpRequestException or authentication pop-ups failing.

  1. Open your project folder in File Explorer.
  2. Right-click inside the folder → Git Bash Here.
  3. Run all git commands from that terminal.
  4. When prompted for credentials, use YOUR_GITHUB_USERNAME and your PAT as the password.

Create remote repo & import in Eclipse (tested approach)
  1. On GitHub, create a new repository (e.g. angulartest).
  2. Add a description.
  3. Add a .gitignore — copy contents from your local template: E:\Categories\Categories\Software_Projects\Development_Environment_Set_Up\github-details\dot_ignore_file.txt and commit it on GitHub (initial commit online).
  4. Copy the clone URL from the green Code button, e.g. https://github.com/YOUR_GITHUB_USERNAME/angulartest.git.
  5. In Eclipse: right-click → Import → Git → Projects from Git → Clone URI.
  6. Paste the URI → Next → Next → Import as general project.
Recommended: This remote-first + Eclipse import flow is the tested approach from the original notes.
Push a new local project to GitHub

Always create an empty repository on GitHub first, then connect your local folder.

  1. Create empty repo on github.com (no README if you already have local files, or accept merge steps below).
  2. Create project directory locally and add a README if starting from scratch.
New project → first push
Customize parameters:
mkdir spring_security_tmplt && cd spring_security_tmplt && echo "# spring_security_tmplt" >> README.md && git init && git add README.md && git commit -m "first commit" && git branch -M main && git remote add origin https://github.com/giventadeveloporg/spring_security_tmplt.git && git pull origin main --allow-unrelated-histories && git push -u origin main
Windows single-line (Git Bash):
mkdir spring_security_tmplt && cd spring_security_tmplt && echo "# spring_security_tmplt" >> README.md && git init && git add README.md && git commit -m "first commit" && git branch -M main && git remote add origin https://github.com/giventadeveloporg/spring_security_tmplt.git && git pull origin main --allow-unrelated-histories && git push -u origin main
Force push (-f): Only use git push origin main -f when you intentionally overwrite remote history (e.g. after creating README on GitHub and rebasing). Avoid on shared branches.
Push an existing local repository

After creating an empty repo on GitHub:

Existing repo → add remote & push
Customize parameters:
git remote add origin https://github.com/giventadeveloporg/java_code_samples_interview.git && git branch -M main && git push -u origin main
Windows single-line:
git remote add origin https://github.com/giventadeveloporg/java_code_samples_interview.git && git branch -M main && git push -u origin main

If origin already exists, use git remote set-url origin https://github.com/… instead of git remote add.

If default branch is master, try git push -u origin master or rename with git branch -M main first.

Eclipse Team / Git workflow

Option A — Create local repo then push

  1. Create repo on GitHub (e.g. dog-app).
  2. In Eclipse: create a new Maven project.
  3. Right-click project → Team → Git → Create Repository (e.g. under C:\Users\YOUR_USER\git).
  4. Right-click → Team → Add to Index (stage files).
  5. Commit, then configure push (see Option B).

Option B — Share project

  1. Right-click project → Team → Share Project.
  2. Check Create project in parent folder of project.
  3. Click Create Repository → Finish.
  4. Open Git Repositories view → your project → Remotes → right-click → Configure Push.
  5. Set URI to https://github.com/YOUR_GITHUB_USERNAME/REPO_NAME.git (e.g. spring_tiles_tmplt).
GitHub CLI — create repo from your PC

You can add a project to GitHub from your machine without opening github.com in the browser (after one-time gh auth login).

winget install GitHub.cli
gh repo create (public or private)
Customize parameters:
cd e:\project_workspace\catholicatenews-in && git init && git add . && git commit -m "Initial commit" && gh auth login && gh repo create catholicatenews-in --public --source=. --push
Windows single-line:
cd e:\project_workspace\catholicatenews-in && git init && git add . && git commit -m "Initial commit" && gh auth login && gh repo create catholicatenews-in --public --source=. --push

If gh repo create … --push fails auth, complete gh auth login, then:

git branch -M main
git push -u origin main
Troubleshooting

non-fast-forward / push rejected

! [rejected] main -> main (non-fast-forward)
hint: Updates were rejected because the tip of your current branch is behind

Fix: Pull remote changes first, then push:

git pull origin main --rebase
git push -u origin main

Or merge: git pull origin main then resolve conflicts and push.

fatal: remote origin already exists

Fix: git remote set-url origin https://github.com/YOUR_GITHUB_USERNAME/REPO_NAME.git

HttpRequestException encountered

Fix: Switch to Git Bash (see above). Ensure HTTPS uses PAT as password, not account password.

Authentication prompts

Username for 'https://github.com'YOUR_GITHUB_USERNAME

Complete authentication in the browser popup if Git Credential Manager opens it.
