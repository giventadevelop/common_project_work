👥 GitHub Organization — Add Seat & Invite Member

Playbook for adding a licensed seat and inviting a new developer to a GitHub Team or Enterprise organization. No credentials or secrets stored here.
Full guide available: See github-org-configuration-guide.html for the complete giventadeveloporg playbook — teams, release_branch_ruleset bypass, repository Write access, and troubleshooting (including the mosc-temp fix).
Prerequisites: You must be an organization owner (or have permission to manage billing and members). The invitee needs a GitHub account linked to the email address you invite. On paid plans, each member who accesses private repositories consumes one seat.

📌 Standard Operating Procedure

  1. Check seat availability — if all seats are used, add a seat before inviting (see below).
  2. Invite the user via the GitHub web UI or gh CLI.
  3. Grant repository access with Write or Maintain so they can commit and merge.
  4. Confirm branch protection rules allow merges to protected branches (e.g. main).
  5. Invitee accepts the email invitation and signs in to GitHub.

Step 1 — Add a seat (licensing)

Required on GitHub Team / Enterprise when you see “No seats are available to invite this user.”

  1. Sign in to GitHub as an organization owner.
  2. Open organization Settings → Licensing (or go directly to):
    https://github.com/organizations/YOUR_ORG/settings/licensing
  3. Review Seats used vs Seats purchased.
  4. Click Add seats (or equivalent) and purchase / allocate one additional seat.
  5. Confirm billing updated before sending a new invitation.

Alternative: free a seat without purchasing optional

If you do not want to buy another seat, remove an inactive member or cancel a pending invitation first.

  • Members: https://github.com/organizations/YOUR_ORG/people
  • Pending invites: https://github.com/organizations/YOUR_ORG/settings/members

Step 2 — Invite a new member

Option A — GitHub web UI

  1. Go to https://github.com/organizations/YOUR_ORG/settings/members
  2. Click Invite member.
  3. Enter the user’s email (e.g. new-developer@example.com) or GitHub username.
  4. Choose role: Member (typical for developers) or Owner (full org admin — use sparingly).
  5. Send the invitation.

Option B — GitHub CLI (gh)

Requires gh auth login as an org owner. Replace placeholders with your values.

gh api orgs/YOUR_ORG/invitations \
  -f email=new-developer@example.com \
  -f role=direct_member

A successful response includes "email", "role": "direct_member", and "failed_at": null. The invitee receives an email to accept.

Expected success response (example shape) reference

{
  "email": "new-developer@example.com",
  "role": "direct_member",
  "created_at": "2026-06-16T00:10:51.000-04:00",
  "failed_at": null,
  "failed_reason": null,
  "invitation_source": "member"
}

Step 3 — Grant commit & merge permissions

Organization membership alone does not grant access to private repositories. Assign repo or team permissions separately.

Per repository (quick)

  1. Open the repo → Settings → Collaborators and teams.
  2. Add the user (or a team they belong to).
  3. Set role to Write or Maintain.

Via a team (recommended for multiple repos)

  1. Organization → Teams → create or select a team (e.g. developers).
  2. Add the new member to the team.
  3. On the team page → Repositories → add repos with Write or Maintain.
Repository roleCan commit?Can merge PRs?
ReadNo (clone/pull only)No
WriteYesYes (if branch rules allow)
MaintainYesYes + limited repo management
AdminYesYes + full repo settings

Step 4 — Branch protection (if merges are blocked)

If main (or default branch) is protected, verify the new member can merge.

  1. Repo → Settings → Branches → edit the protection rule.
  2. Ensure Write / Maintain users or their team are allowed to push or merge pull requests.

🚨 Known errors & fixes

“No seats are available to invite this user” HTTP 422

All purchased seats are in use (active members and sometimes pending invitations).

Fix: Add a seat at /organizations/YOUR_ORG/settings/licensing, or remove an unused member / cancel a pending invite, then retry. 
gh api orgs/YOUR_ORG/invitations \
  -f email=new-developer@example.com \
  -f role=direct_member

Invite sent but user cannot push or merge permissions

They accepted the org invite but have no repository role yet, or branch protection blocks merges.

Fix: Grant Write or Maintain on the target repo(s) or via a team. Check branch protection on main.

🔧 Useful CLI commands (diagnostics)

Replace YOUR_ORG and usernames. Do not commit tokens or PATs to documentation.

# List current org members
gh api orgs/YOUR_ORG/members --paginate -q ".[].login"

# List pending invitations
gh api orgs/YOUR_ORG/invitations

# Cancel a pending invitation (use id from invitations list)
gh api -X DELETE orgs/YOUR_ORG/invitations/INVITATION_ID

# Remove a member (use GitHub username, not email)
gh api -X DELETE orgs/YOUR_ORG/members/USERNAME

👤 What the new member must do

  • Have (or create) a GitHub account using the invited email address.
  • Accept the organization invitation from email or from https://github.com/notifications.
  • Configure Git access (SSH key or HTTPS with a personal access token).
  • Clone repositories they were granted access to.

🔒 Security notes

  • Do not store passwords, personal access tokens, or API keys in this document or in git.
  • Grant Owner only to trusted administrators.
  • Prefer team-based access over adding users to many repos individually.
  • Review /organizations/YOUR_ORG/settings/billing periodically for unused seats.
